# Code review сгенерированного ИИ-кода (задание повышенной сложности №2)

Формат: **что сгенерировал ИИ → в чём проблема → как исправил**.

## 1. Запись файлов без async-обвязки

**ИИ предложил:** в примере из методички синхронная запись `open(...); f.write(await file.read())` внутри async-эндпоинта.

**Проблема:** блокирует event loop; для продакшена нужны `aiofiles` или фоновые задачи.

**Исправление:** в данном проекте файловая загрузка не используется; для API оставлены только JSON-тела. Если добавится upload — вынести в async-совместимый слой.

## 2. Отсутствие каскадного удаления для `User → Board`

**ИИ предложил:** связь `User.boards` без `cascade`, удаление пользователя-админа ломало FK или оставляло «осиротевшие» записи.

**Проблема:** нарушение целостности БД при `DELETE` пользователя.

**Исправление:** добавлен `cascade="all, delete-orphan"` для `boards` и `comments` у пользователя (`models.py`).

## 3. Имя модели `Column` в SQLAlchemy

**ИИ предложал:** класс `Column`, пересекающийся по имени с `sqlalchemy.Column`.

**Проблема:** путаница импортов, риск тонких багов.

**Исправление:** сущность переименована в `BoardList`, таблица `board_lists`, поле связи `list_id`.

## 4. Неточная обработка `PUT` для `due_date`

**ИИ предложал:** смешанную проверку `payload.due_date is not None` и `model_fields_set`, что усложняло сброс даты в `null`.

**Проблема:** неочевидная семантика частичного обновления.

**Исправление:** единый путь через `payload.model_dump(exclude_unset=True)` и `setattr` по полям (`routers/cards.py`).

## 5. CI без закреплённой версии Python

**ИИ предложал:** абстрактный `python-version: '3.x'` или использование системного Python раннера.

**Проблема:** на части машин (например, Python 3.14 без MSVC) зависимости не собираются из исходников.

**Исправление:** в GitHub Actions зафиксирован **Python 3.12**; в README явно указано требование к версии / Docker.

## 6. (Дополнительно) Комментарий в PR на каждый push

**ИИ предложал:** триггер `synchronize` для каждого коммита в PR.

**Проблема:** спам комментариями, сложнее читать обсуждение.

**Исправление:** workflow ограничен событиями `opened` и `reopened` (см. `.github/workflows/ai-pr-summary.yml`).
