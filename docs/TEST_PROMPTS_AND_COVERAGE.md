# Задание повышенной сложности №7 — промпты для unit-тестов и покрытие

## Цель

Достичь **не менее 90%** покрытия строк пакета `taskboard` при запуске:

```bash
set PYTHONPATH=src
pytest tests/ -q --cov=taskboard --cov-report=html --cov-fail-under=90
```

В CI проверка включена в `.github/workflows/ci.yml` (Ubuntu, Python 3.12).

## Система промптов (шаблоны)

### A. Граничные случаи домена (сроки карточек)

> Для FastAPI-приложения «доски и карточки» напиши pytest-тест: создай доску, колонку и карточку с `due_date` в прошлом (timezone-aware UTC). Вызови `GET /analytics/boards/{id}` и утверждай, что `overdue_cards == 1`. Используй TestClient и фикстуру БД из conftest.

### B. Безопасность и авторизация

> Добавь тесты: запрос к `/auth/me` без заголовка `Authorization` возвращает 401; попытка доступа к чужой доске по id возвращает 404; обычный пользователь не может вызвать `/admin/users` (403).

### C. Pydantic-валидация

> Сгенерируй тесты на `UserCreate` и `BoardCreate`: пароль короче 8 символов и пустой `title` доски должны давать `ValidationError`.

### D. Изоляция и порядок тестов

> Убедись, что `conftest.py` на каждый тест пересоздаёт схему SQLite в памяти (StaticPool), патчит `taskboard.database.engine` и `SessionLocal`, и сбрасывает `app.dependency_overrides` после теста.

## Итог

- Сгенерированные по шаблонам тесты лежат в `tests/`.
- Порог покрытия **90%** задан ключом `--cov-fail-under=90` в CI (локально — тот же флаг).
- HTML-отчёт (при локальном запуске): `htmlcov/index.html`.

## Примечание про `coverage.py`

Артефакт `htmlcov/` в репозиторий не коммитится (генерируется локально/в CI). Для отчёта преподавателю достаточно скриншота из CI или локального `htmlcov`.
