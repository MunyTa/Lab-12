# Промпты, использованные при выполнении ЛР №12 (вариант 8)

Ниже — формулировки, с которыми работал ИИ-ассистент. Сохранены удачные шаблоны; для сдачи важно показать и неудачные итерации — их кратко отражено в `docs/CODE_REVIEW.md`.

## 1. Каркас FastAPI + предметная область (Trello-клон)

> Ты — senior Python-разработчик. Спроектируй REST API на FastAPI для системы управления задачами (доски, колонки на доске, карточки с описанием и сроком, комментарии к карточкам). Нужны: SQLAlchemy-модели с каскадным удалением в разумных местах, Pydantic v2-схемы, JWT-аутентификация (регистрация + OAuth2 password flow), разграничение владельца доски и администратора (`is_admin`). Добавь эндпоинт аналитики по доске: число карточек, просроченных по `due_date`, число комментариев, число колонок. Код разбей на модули (`routers`, `schemas`, `security`, `deps`), избегай монолитных функций на сотни строк.

## 2. Тесты (pytest + TestClient)

> Сгенерируй интеграционные тесты на pytest для приложения выше. Используй отдельную in-memory SQLite через monkeypatch на `engine` и `SessionLocal`, чтобы тесты не трогали файл БД. Покрой: регистрацию, логин, CRUD досок, списки и карточки, перенос карточки между колонками, комментарии, аналитику, запрет доступа к чужой доске, админ-маршруты после выставления `is_admin` в БД.

## 3. GitHub Actions + комментарий в PR

> Напиши GitHub Actions workflow: на событие `pull_request` (opened/reopened) получить diff между base и head SHA, при наличии `OPENAI_API_KEY` вызвать OpenAI Chat Completions (`gpt-4o-mini`) и опубликовать комментарий к PR через GitHub REST API. Если ключа нет — всё равно оставь комментарий с усечённым diff и пояснением, что нужен секрет. Реализацию вынеси в отдельный скрипт `scripts/pr_ai_comment.py` на стандартной библиотеке Python.

## 4. Docker

> Создай Dockerfile на базе `python:3.12-slim-bookworm`, установи зависимости из `requirements.txt`, выставь `PYTHONPATH=/app/src`, точка входа — `uvicorn taskboard.main:app`. Добавь `docker-compose.yml` с томом для SQLite.
